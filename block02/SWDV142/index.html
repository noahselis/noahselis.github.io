What is a Blockchain?

A blockchain is a peer to peer distributed network of folks volunteering their computers to act as a sort of decentralized database.

These computers are called “nodes”. All of these nodes contain a copy of the code and all of the data that is on the blockchain. All of this information is bundled together as it is uploaded as “blocks” and and these blocks are “chained” together.

Changing any one of these blocks, because they are all chained together, requires the consensus of all of the nodes on the chain making it a very secure and transparent system.

The many benefits of programming on a blockchain

Transparency

In order for something to be stored on a blockchain, it must have "proof of work" that involves solving a difficult problem with brute force trial and error that uses a lot of computing power.

Trustless

You don't need a bank or government verifying everything that you do. Everything is peer to peer.

Ownership

When you create something on YouTube, YouTube ultimately owns what you created. When you create something and put it on the blockchain, it's yours and there are mechanisms in place that ensure only you can access it.

Resists Censorship

With no centralized entity controlling information on the blockchain, it's very difficult for an individual or government to restrict access to the dapp.

Smart Contracts

Smart Contracts are Reusable snippets of code that have been uploaded to and are executed by the network. They’re kind of like a vending machine: A user wants to interact with the smart contract so they give it the necessary parameters and it spits out the result of it’s internal computation. That result could be simply creating and assigning ownership of a digital asset (minting an NFT)

The Ethereum Virtual Machine

The Ethereum Virtual Machine is a canonical computer whose state everyone on the network agrees on. This machine can perform arbitrary computations based on requests from nodes. This causes a change in EVMs "state". This state is agreed upon by everybody using the network because EVM relies on all of the individual nodes to perform these computations.
